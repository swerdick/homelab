# Minecraft (Fabric) artifacts published to Harbor as OCI blobs by
# publish-mc-mods.sh, and pulled onto eregion by ansible/playbooks/install-fabric-mc.yaml.
#
# Format (pipe-delimited, one artifact per line):
#   name | tag | filename | upstream_url | sha512
#
# - name/tag  -> Harbor coordinate:  harbor.vingilot.internal/minecraft/<name>:<tag>
#                (OCI tags can't contain '+', so '+' in versions is written as '-')
# - filename  -> canonical on-disk name; the file is renamed to this before push,
#                so `oras pull` restores exactly this name into the mods/ dir.
#                MUST match the `filename:` in ansible/host_vars/eregion/main.yaml.
# - sha512    -> upstream file hash; verified before push AND after pull. Leave
#                EMPTY to skip pre-verify and have the script print the computed
#                sha512 instead (used for the Fabric launcher jar, which upstream
#                publishes with no checksum — paste the printed value into
#                host_vars `minecraft_launch_jar.sha512`).
#
# Keep this file and host_vars/eregion/main.yaml in sync — both derive from the
# same Modrinth release data. Bumping a mod = edit both, re-run this script,
# re-run the playbook. See runbooks/fabric-upgrade.md.
